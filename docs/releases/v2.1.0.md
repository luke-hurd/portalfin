# portalfin v2.1.0

Brings back the **ambient screensaver** (rebuilt fully native) and polishes the
**"Now playing" card** the Portal shows on its home screen.

The ambient screensaver was a v1.x feature that lived in the WebView layer, so the
v2.0 native rebuild (which moved home/library/detail to Compose) left it stranded:
idling just showed the Portal's own clock-on-black. v2.1 reimplements it as a
native Compose overlay that runs on every screen.

## What's new

### Native ambient screensaver
- **Idle slideshow** — after ~60s with no interaction, a fullscreen slideshow of
  random cover art from your library fades in.
- **Slow Ken Burns** — each backdrop gently zooms and drifts (direction alternates
  per slide) for subtle, premium motion. Slides start exactly fullscreen, so the
  pan never exposes an edge.
- **Gap-free crossfade** — the incoming backdrop fades in *on top* of the current
  one (which stays fully opaque underneath), so the background never shows through
  between slides.
- **Clock + date + title art** — a large time/date in Inter, and the title's logo
  art (movie/series) in the corner, with a legibility scrim.
- **Edge-to-edge** — the Portal's system back/home OSD band is hidden while the
  screensaver is up, then restored on dismiss.
- **Stays out of the way** — any touch dismisses it instantly; it never engages
  while a video is playing or while the app is backgrounded (so you won't return
  to a running screensaver).

### "Now playing" card branding
- The media card the Portal surfaces on its home screen while you're watching now
  shows the **portalfin logo** instead of the Jellyfin one.
- It features the title's **portrait poster** as artwork — the series poster for
  episodes (whose own image is a 16:9 still), the movie's poster for films.

## Sideload

The download link always points at the newest release:

1. `adb devices` confirms your Portal is connected (ADB enabled in Settings → Debug)
2. Download **[portalfin.apk](https://github.com/luke-hurd/portalfin/releases/latest/download/portalfin.apk)**
3. `adb install -r portalfin.apk` (the `-r` upgrades in place)

## Known limitations

- Weather on the ambient screen isn't wired up yet (clock + date are) — planned.
- Person/actor pages and the admin dashboard still open the WebView fallback.
- The top ~64px is the Portal system OSD overlay; interactive UI is kept below it.
