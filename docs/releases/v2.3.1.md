# portalfin v2.3.1

Makes the v2.3.0 self-update actually work.

## Fixes

- **Stable signing key.** Releases were signed with throwaway debug keys that
  differed between machines/CI builds, so installing an update failed with "App
  not installed" (Android blocks in-place updates when the signature changes).
  All builds now share one committed debug keystore, so self-update — and
  `adb install -r` — work going forward.
- **Self-update permission flow.** If portalfin isn't yet allowed to install apps,
  the update prompt now sends you straight to its own "install unknown apps"
  toggle (instead of a dead-end "not allowed from this source" dialog). Enable it
  once, tap Update again, done.

> One-time note: updating to v2.3.1 from an older build may fail to install (old
> signature) — if so, uninstall and sideload v2.3.1 once. Updates from v2.3.1
> onward install cleanly and keep your login/settings.

## Sideload

The download link always points at the newest release:

1. `adb devices` confirms your Portal is connected (ADB enabled in Settings → Debug)
2. Download **[portalfin.apk](https://github.com/luke-hurd/portalfin/releases/latest/download/portalfin.apk)**
3. `adb install -r portalfin.apk` (use `adb install -r -d`, or uninstall first, if it refuses due to the older signature)
