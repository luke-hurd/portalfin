# portalfin v2.3.2

Fixes the screensaver and the misleading update message.

## Fixes

### Ambient screensaver
- **Only runs when signed in.** The screensaver pulls artwork from your Jellyfin
  server, so it no longer activates on the Connect/Login screens (where there's no
  server yet) — which is what produced an empty, image-less screensaver.
- **No more blank screensaver when logged in.** It now (re)loads artwork each time
  it engages and retries if a fetch comes back empty, instead of sitting on just
  the clock. (Root cause: the idle timer armed before login finished and never
  re-armed once you were signed in.)

### Updates
- **No more false "App not installed."** Self-update now uses Android's modern
  package-installer session, which reports the real result — so a successful
  update no longer shows a misleading "App not installed" screen afterward. On
  success the update just completes; failures show the actual reason.

## Sideload

The download link always points at the newest release:

1. `adb devices` confirms your Portal is connected (ADB enabled in Settings → Debug)
2. Download **[portalfin.apk](https://github.com/luke-hurd/portalfin/releases/latest/download/portalfin.apk)**
3. `adb install -r portalfin.apk`

From v2.3.1 onward, future versions update themselves from inside the app.
