# portalfin v1.0.0

First public release. Native portalfin shell wrapping jellyfin-web on Facebook Portal devices (codename "aloha").

## What's in it

- Native Compose `LoginScreen` calling the Jellyfin SDK directly, then seeding the WebView's localStorage so it lands authenticated on `/home`
- Custom `#portalfin-header` replacing jellyfin-web's chrome (51px tall, sticky under Portal's reserved overlay band)
- `portalfin-restyle.js` injected on every page: Portal palette, kiosk chrome hiding, sign-out detection, SPA-route re-injection
- Portal-specific battery-optimization snackbar suppression
- Custom launcher icon with proper padding for Portal's launcher mask
- PortalFin loader (wordmark + spinner) between splash and WebView
- Full sign-out lifecycle: jellyfin-web logout clears native `UserEntity` and routes back to Compose `LoginFragment`

## Verified on

- Portal Gen 1 ("aloha"), firmware build `1041515800015050` (Oct 2025)

Other Portal models (Gen 2, Mini, Plus, Go, TV) likely work but are untested.

## Sideload

1. `adb devices` confirms your Portal is connected (with ADB enabled in Settings -> Debug)
2. Download `portalfin-v1.0.0.apk` from the assets below
3. `adb install portalfin-v1.0.0.apk`
4. Launch from Portal's Apps screen

## Known limitations

- Default host field is empty — type your Jellyfin server's URL on first launch
- The native ExoPlayer chrome is unmodified upstream — only used if you select "Integrated player" in Settings; the default web player is unaffected
- No native Portal home grid yet (the home screen is jellyfin-web rendered inside the WebView with Portal restyling)
