# portalfin v2.3.3

A startup-stability fix.

## Fix

- **No more "Required value baseUrl is null" on the home screen.** On a cold start
  — especially after Android kills and restores the app from the background — the
  home screen could try to load library data a split second before the saved
  server connection finished initializing, and get stuck on a red error. The home
  screen now waits for the connection to be ready before loading, so it comes up
  normally. (Your data/login were never affected — it was a timing issue.)

## Sideload

The download link always points at the newest release:

1. `adb devices` confirms your Portal is connected (ADB enabled in Settings → Debug)
2. Download **[portalfin.apk](https://github.com/luke-hurd/portalfin/releases/latest/download/portalfin.apk)**
3. `adb install -r portalfin.apk`

Already on v2.3.1+? portalfin can update itself from inside the app.
