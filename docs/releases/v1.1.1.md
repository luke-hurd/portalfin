# portalfin v1.1.1

Bug fixes + polish on top of v1.1.0. Same auth flow, same sideload path.

## Critical fix

- **Tapping a movie poster no longer boots you back to home.** The
  `kioskizeChrome` redirect regex was matching `serverId=` in detail-page
  URLs as if it were the `/server` admin route. Fixed by anchoring the
  regex on the path part only. **This affected every detail navigation in
  v1.0.0 and v1.1.0** — if you couldn't open any movies before, this is
  why.

## Other improvements

- **Page title in the header back-arrow.** Library and detail pages now
  show `< Movies` / `< 10 Things I Hate About You` etc. next to the back
  button, like proper kiosk apps do.
- **Ambient slideshow now renders backdrop images.** A CSS specificity
  bug (opacity:0!important winning over inline style) was keeping every
  Jellyfin backdrop invisible. Switched to a class-based opacity toggle.
  Verified with Willow (1988) backdrop visible behind the clock.
- **Wordmark +20% larger** (175→210 wide, 40→48 tall). Header height
  bumped 58→68 to fit. Page padding follows.
- **Frosted-glass header.** `backdrop-filter: blur(20px) saturate(180%)`
  with a translucent rgba background so scrolled content is fuzzy-visible
  underneath. The time-of-day tint applies through the glass.

## Process improvements

- **Pre-merge testing protocol** is now codified in
  [GITFLOW.md](https://github.com/luke-hurd/portalfin/blob/main/GITFLOW.md).
  Three-view regression sweep, end-to-end navigation test, "tap missed"
  is a test failure not a free pass. Future feature branches won't merge
  without visual proof on the actual device.

## Sideload

```bash
adb install -r portalfin-v1.1.1.apk
```

The `-r` upgrades over v1.1.0 in place.
